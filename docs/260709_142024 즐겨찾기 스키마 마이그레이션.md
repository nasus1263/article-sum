# 즐겨찾기(favorited_at) 추가 — Supabase 마이그레이션

## 배경

Archive 탭 각 항목에 즐겨찾기 별표 버튼을 추가하고, 즐겨찾기한 항목만 모아보는 Favorites 탭을 구현하기 위해 `contents` 테이블에 즐겨찾기 시각 컬럼을 추가.

- `null` = 즐겨찾기 아님
- `not null`(즐겨찾기 등록 시각) = 즐겨찾기됨 — Favorites 탭에서 이 값 기준 내림차순(최근 즐겨찾기한 순) 정렬

## 스키마 변경

`data`(jsonb)가 아닌 실제 컬럼으로 추가 — 정렬/필터에 인덱스 활용 가능하도록.

```sql
alter table contents
  add column favorited_at timestamptz;
```

기존 행은 전부 `null`(즐겨찾기 아님)로 채워짐. 별도 백필 불필요.

## 애플리케이션 변경

- `electron/db.js`: `rowToRecord`에 `favoritedAt` 매핑 추가, `listByStatus`/`getContent` select 목록에 `favorited_at` 추가, `setFavorite(id, favorited)` 추가(등록 시 `now()`, 해제 시 `null`)
- `electron/main.js`: `contents:set-favorite` IPC 핸들러 추가
- `electron/preload.js`: `window.api.setFavorite(id, favorited)` 노출
- `src/types/global.d.ts`: `ContentRecord.favoritedAt: string | null` 추가
- `src/components/FeaturedArchiveList.tsx`: 항목 우측 상단 별표 버튼(`onToggleFavorite`) 추가
- `src/pages/Archive.tsx`: `favoritesOnly` prop 추가 — true면 `favoritedAt`이 있는 항목만, 최근 즐겨찾기 순으로 정렬 후 기존 검색/카테고리/폴더 필터 적용. Archive 탭과 Favorites 탭이 이 컴포넌트를 공유(모듈화)하며 `favoritesOnly` 값만 다름
- `src/App.tsx`: Favorites 탭에서 `<Archive variant="favorites" favoritesOnly />` 렌더

## 확인 필요

- 컬럼 추가는 기본값 없는 nullable 추가라 락 부담 적음(테이블 재작성 없음) — 실 운영 데이터 규모에서도 안전.
