# 이미지 목록(images) 전환 — Supabase 마이그레이션

## 배경

백엔드 `/process` 응답이 이미지 단일 필드(`image: str`)에서 목록(`images: list[str]`)으로 바뀜에 따라 프론트/Electron도 `data.thumbnail` → `data.images: string[]`로 전환.

- `electron/main.js`: `data.images = result.images ?? []`
- `src/types/global.d.ts`: `ContentRecord.data.thumbnail` → `images?: string[]`
- `src/pages/Pending.tsx`, `src/pages/Archive.tsx`, `src/pages/Chat.tsx`: 목록/헤더에는 `data.images[0]`(첫 이미지)만 미리보기로 표시
- `src/pages/ArchiveDetail.tsx`: 기사 상세 페이지에서 `data.images` 전체를 순서대로 표시

## 스키마 변경 여부

`contents.data`는 `jsonb` 컬럼이라 `ALTER TABLE` 불필요 — 필드명/형태 변화는 애플리케이션이 쓰는 JSON 값 자체의 변화일 뿐. 신규로 처리되는 행은 그대로 `data.images` 배열로 저장됨.

단, **기존에 이미 저장된 행**은 `data.thumbnail`(문자열)만 갖고 있고 `data.images`가 없어서 상세 페이지에 사진이 안 보임. 아래 SQL로 1회성 백필 필요.

## SQL 마이그레이션 (1회 실행)

```sql
-- data.thumbnail(string) → data.images(string[])로 백필, 기존 thumbnail 키는 제거
update contents
set data = (data - 'thumbnail') || jsonb_build_object(
  'images',
  case
    when data ? 'thumbnail' and data->>'thumbnail' is not null then jsonb_build_array(data->'thumbnail')
    else '[]'::jsonb
  end
)
where data ? 'thumbnail' and not (data ? 'images');
```

- `data ? 'thumbnail'`: thumbnail 키가 있는 행만 대상
- `not (data ? 'images')`: 이미 images로 전환된 행은 건드리지 않음(재실행 안전)
- thumbnail 값이 null이었던 행은 `images: []`로 정리

## 확인 필요

- 실행 전 `contents` 테이블 백업(또는 Supabase Point-in-time restore 가능 여부) 확인 권장 — jsonb 컬럼 전체를 덮어쓰는 UPDATE라 되돌리기 어려움.
